## Template Fields Guide

### ROLE
Examples:
- "senior full-stack engineer"
- "backend architect"
- "frontend specialist"
- "DevOps engineer"

### FEATURE_NAME
Clear, descriptive name (e.g., "User Preferences", "Notification System", "Audit Log")

### BRIEF_DESCRIPTION
One sentence describing what the feature does

### KEY_TECHNICAL_REQUIREMENT
Main technical constraint or approach (e.g., "database-agnostic adapter", "event-driven architecture")

### PRIMARY_GOAL
Specific, actionable goals. Use numbered list for main goals, bullets for sub-requirements.

### TECHNICAL_SECTION
Sections for:
- Database schemas
- API endpoints
- Configuration
- Data structures
- Integration points

### Code Examples
Include:
- Scripts for automation
- API endpoint definitions
- Type definitions
- Configuration examples

### NOTES
Important considerations:
- Security
- Performance
- Limitations
- Future enhancements
- Compliance requirements
